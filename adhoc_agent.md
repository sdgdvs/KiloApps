# Ad-Hoc Agent Guide

Welcome to the KiloApps project! If you are an ad-hoc agent brought in to fix a one-off issue, implement a specific feature, or answer a question, please follow the rules in this document to ensure you don't step on the toes of the autonomous agent fleet and ensure your work gets properly deployed.

## 1. The Multi-Agent Environment
This repository is continuously modified by a fleet of 5 autonomous agents operating on overlapping schedules. They are constantly pushing and pulling code.
- **Rule:** **ALWAYS `git pull`** before you start reading or editing any files to ensure you have the latest changes. 
- **Rule:** If you encounter merge conflicts when pushing, use `git pull --rebase`, resolve conflicts conservatively (preferring remote code for things you didn't write), and push again.

## 2. CI/CD & Deployment (Crucial)
The website (`kiloapps.web.app`) is automatically rebuilt and hosted by Firebase **only when code is pushed to the `main` branch on GitHub.**
- **Rule:** Local changes to the web files will NOT go live on their own. 
- **Rule:** Once you have fixed the issue, you **MUST** `git add`, `git commit -m "..."`, and `git push origin main`. If you don't push, the user won't see your changes!

## 3. The Dual-Target App Model
Almost every KiloApp exists in two forms. When fixing a bug in an app's logic, try to fix it in both places if applicable.
1. **Native Windows Application:** Standalone C/C++ or ASM executable. Located in `K[Name]/` (e.g., `KCalc/main.c`). Compiled using the local `build.bat` script.
2. **Web (HTML5) Application:** A single `.html` file containing all logic, inline CSS, and HTML. Located in `KiloOS/public/apps/` or `KiloOS_Server/public/apps/`. **No external CSS/JS imports are allowed inside individual web apps.**

## 4. Size Constraints & Philosophy
- **Rule:** **No individual KiloApp may exceed 999 kilobytes** in size (both for the native executable and the web HTML file). Every kilobyte is accounted for. Keep your code minimal, vanilla, and dependency-free.

## 5. Shared Files & Ownership
- **`KiloOS/src/App.jsx`**: This is the main OS shell. It is owned by the UX Agent. Only modify this if you absolutely have to (e.g., fixing an app's registry entry or adding a new app to the `APPS` array). If you do, `git pull`, make the minimal change, and commit/push *immediately*.
- **`KiloOS/src/index.css`**: Do NOT edit this file.
- **Plan Files** (e.g., `master_plan.md`, `app_work_plan.md`, `kiloos_ux_plan.md`): These belong to the persistent agents. **Do not edit them.**

## 6. Testing Your Changes
- **Web changes:** You can test HTML files directly in a browser. If you modify KiloOS React files, `cd KiloOS` and run `npm run build` to verify it compiles.
- **Native changes:** Run the `build.bat` in the specific app's folder to ensure your C code compiles without errors. (Note: MSVC `cl` might not be in your standard path, but ensure the code is syntactically correct).

## 7. Agent Efficiency Tools (CRITICAL)
- **`rg` (ripgrep), `fd`, and `uv` are installed globally.**
- Always use `rg` to search for text across files instead of `grep` or `findstr`.
- Always use `fd` to find files instead of `ls`, `dir`, or `Get-ChildItem`.
- Always use `uv` for Python package management instead of `pip`.
